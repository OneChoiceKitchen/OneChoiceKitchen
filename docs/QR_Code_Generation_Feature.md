# QR Code Generation Feature Documentation

## Overview
This document explains the automatic QR Code Generation feature implemented within the Admin Portal of the OneChoiceKitchen application. It provides end-to-end details on how the digital menu URLs are constructed, how the QR codes are generated visually on the client side, and how they can be downloaded as printable PNG images.

## Supported Contexts
The QR Code generation operates in two primary contexts:
1. **Branch Level (Generic Menu)**: Generates a QR code for a specific branch without assigning orders to a table.
2. **Table Level (Dine-in Menu)**: Generates a distinct QR code for each physical table, automatically linking orders to that specific table.

---

## 1. URL Construction

The core functionality relies on generating a predictable URL structure that the customer-facing digital menu application will consume. 

### Branch Level URL
When a generic branch QR code is generated, the URL includes the `branchId`.
```typescript
const getBranchQrUrl = (branch: Branch) => {
  return `${window.location.origin}/menu?branchId=${branch.id}`;
};
```
*Note: `window.location.origin` is used as a placeholder. In a production environment where the customer-facing app operates on a different domain (e.g., `https://menu.onechoicekitchen.com`), this prefix should be updated via environment variables.*

### Table Level URL
For table-specific QR codes, the URL is extended to include both the `branchId` and the `tableId`.
```typescript
const getTableQrUrl = (table: any) => {
  return `${window.location.origin}/menu?branchId=${table.branchId}&tableId=${table.id}`;
};
```

## 2. Visual Generation using `qrcode.react`

We utilize the `qrcode.react` library to dynamically generate the visual QR code within the React component tree.

### Implementation Details
- **Component**: `<QRCodeSVG />` is used instead of `<QRCodeCanvas />`. SVG is preferred as it scales infinitely without quality loss, ensuring sharp edges when viewing.
- **Props**:
  - `value`: The target URL generated by the functions above.
  - `size={220}`: Renders a large, easily scannable code.
  - `level="H"`: High Error Correction Level (up to 30% damage tolerance). This is critical for physical menus that might get scratched, stained, or partially obscured.
  - `includeMargin={true}`: Ensures a quiet zone around the QR code, which is required by most camera scanners.
  - `fgColor="#0f172a"`: Uses the application's primary dark slate color instead of stark black for a softer, integrated look.

```tsx
<QRCodeSVG 
  id={`qr-code-${entity.id}`} // Unique ID required for the download step
  value={url} 
  size={220} 
  level="H" 
  includeMargin={true}
  fgColor="#0f172a"
/>
```

## 3. SVG to PNG Download Logic

Because `<QRCodeSVG />` renders as an inline SVG element, users cannot simply "Right Click -> Save Image As" directly in all browsers, and SVGs are sometimes difficult for non-technical users to print. 

To solve this, we implemented a custom download function that dynamically transforms the SVG into a PNG image.

### Transformation Steps:
1. **Target the SVG**: We assign a unique HTML `id` to the `<QRCodeSVG />` element and query it using `document.getElementById()`.
2. **Serialize to XML**: `XMLSerializer().serializeToString(svg)` converts the DOM SVG node into a raw XML string.
3. **Canvas Rendering**: 
   - A hidden `<canvas>` element is created.
   - A padding of 40px (20px on all sides) is added to the canvas size to ensure it prints well.
   - The canvas is filled with a solid `white` background (since SVGs are transparent by default).
   - An `Image` object is created and its source is set to a base64-encoded version of the SVG XML.
   - Once the image loads, it is drawn onto the canvas using `ctx.drawImage()`.
4. **Data URL Conversion**: The canvas is converted to a PNG Data URI using `canvas.toDataURL('image/png')`.
5. **Trigger Download**: A temporary `<a>` tag is created, assigned the PNG Data URI as its `href`, given a dynamic `download` filename (e.g., `Table-12-QR.png`), and programmatically clicked.

```typescript
const downloadQR = (table: any) => {
  const svg = document.getElementById(`qr-code-${table.id}`);
  if (!svg) return;
  const svgData = new XMLSerializer().serializeToString(svg);
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');
  const img = new Image();
  img.onload = () => {
    canvas.width = img.width + 40; // Add padding
    canvas.height = img.height + 40;
    if (ctx) {
      ctx.fillStyle = 'white';
      ctx.fillRect(0, 0, canvas.width, canvas.height);
      ctx.drawImage(img, 20, 20); // Draw image with padding
      const pngFile = canvas.toDataURL('image/png');
      const downloadLink = document.createElement('a');
      downloadLink.download = `Table-${table.tableNumber}-QR.png`;
      downloadLink.href = `${pngFile}`;
      downloadLink.click();
    }
  };
  // unescape(encodeURIComponent()) ensures proper handling of special characters
  img.src = 'data:image/svg+xml;base64,' + btoa(unescape(encodeURIComponent(svgData)));
};
```

## 4. UI/UX Flow

### In `BranchesAdmin.tsx`
- When the **Enable QR Digital Menu** toggle is active, a **View QR** button appears next to the **Manage** button on the branch card.
- Clicking **View QR** opens a centered modal with a blurred overlay.
- The modal displays the generated QR code, the raw URL, and a prominent blue **Download Menu QR Code** button.

### In `TablesAdmin.tsx`
- The manual `qrCodeUrl` input field has been removed from the Add/Edit Table form.
- Instead, the form displays an informational block stating that the QR code is generated automatically.
- On each table card, a **View QR** button is provided. 
- Clicking it opens a similar modal tailored to that specific table, offering a **Download QR Code PNG** button.

## Future Considerations
- **Bulk Download**: In the future, a feature could be added to generate and download a `.zip` file containing the QR codes for all tables in a branch at once.
- **Custom Branding**: The download logic could be extended to draw the Restaurant's Logo into the center of the QR Code or add branded headers/footers to the generated PNG image before downloading.
